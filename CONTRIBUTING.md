# Contributing

Thank you for considering contributing to Laravel Alias Manager.

## Development Flow

- Use `develop` as the integration branch.
- Create focused branches from `develop`.
- Open pull requests into `develop`.
- Keep commits small and descriptive.
- Use Conventional Commit messages.

## Branch Naming

- `feat/*` for new features
- `fix/*` for bug fixes
- `docs/*` for documentation
- `test/*` for tests
- `refactor/*` for internal improvements
- `ci/*` for automation
- `chore/*` for maintenance

## Commit Examples

```text
chore: add repository identity
feat(installer): add install command
fix(shell): prevent duplicate managed block
docs(readme): improve installation guide
test(installer): cover backup flow
```

## Quality

Before opening a pull request, make sure the relevant checks pass locally.

Detailed commands will be added after the package tooling is installed.
